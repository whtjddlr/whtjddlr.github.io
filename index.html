PlanMerge AI 기획서 병합 협업 도구

팀원들이 각자 AI로 기획서 초안을 만들면, 병합 과정에서 “누구 의견이 왜 채택됐는지”가 사라집니다. PlanMerge는 문서를 생성하는 도구가 아니라 공동 기획의 의사결정 흐름을 보존하는 도구로 포지셔닝해, 채택된 의견·제외된 대안·충돌 지점·원문 출처를 섹션별로 추적합니다.

역할 — 기획·스펙 문서·ERD 설계·개발·배포 단독 수행
  • 문제 정의 — “LLM에 초안을 다 넣으면 그럴듯한 문서는 나오지만, 반영·제외·충돌·근거라는 협업의 핵심 질문이 남는다”에서 출발
  • 품질 게이트 — 섹션 커버리지·출처 커버리지·선택지 추적성을 점수화해, 내보내기 전에 병합 품질을 확인
  • 검증 샘플 설계 — PM·디자이너·개발자·세일즈·마케터 관점의 초안 13개로 샘플을 구성하고, MVP 범위 충돌 1건을 의도적으로 심어 제품 가치가 바로 드러나게 함
  • 문서 선행 — v0.1 스펙 문서와 ERD를 먼저 작성한 뒤 개발 착수

CARCH 카드 소비 코치 · 2인 팀

카드 혜택 조건은 사람이 직접 비교하기 어렵습니다. CARCH는 결제내역·보유 카드·소비계획을 함께 분석해 “지금 가진 카드 활용”과 “새 카드 발급”을 분리해 제안합니다 — 연회비를 월 환산해 비교하고, 보유 카드 대비 추가 이득이 있는 후보만 발급 추천에 올리는 것이 제품 원칙입니다.

역할 — 백엔드 API · 데이터 모델링 · 혜택 데이터 정제 · 추천 로직 · AI 연동 · 배포 구조
  • AI 역할 원칙 — 생성형 AI가 숫자를 새로 만들지 않고, 규칙 기반 계산 결과를 사용자가 이해하기 쉽게 설명만 하도록 역할을 분리
  • 채널 운영 설계 — 같은 코드베이스에서 환경변수·DB 스키마를 분리해 App Store 기준 서비스 채널과 발표용 자동 로그인 데모 채널을 함께 운영
  • 스토어 요건 완수 — Capacitor iOS 셸, Keychain 토큰 저장, 계정 삭제, 개인정보 처리방침 등 App Store 검수 요건 충족

KOK 약속 장소 추천 · 온라인 추첨

  • 공평함의 수치화 — 참가자별 대중교통·자동차 이동 시간을 NAVER Maps + ODsay 실경로로 보정해, 이동 시간 편차 기준으로 후보를 좁힘

  • 장애 대비 — AI 후보 재정렬을 GMS·OpenAI·Upstage 폴백 체인으로 묶어 한 provider가 죽어도 동작

지도 앱은 “지금 출발하면 언제 도착하는지”를 알려줍니다. BBARU는 질문을 뒤집었습니다 — “그 시각에 도착하려면 언제 출발해야 하는가?” 면접·공연처럼 도착 시각이 중요한 상황에서 출발 시각 추천과 이동 중 행동 지시까지 제공하는 PWA입니다.

역할 — 기획 · 설계 · 개발 · 테스트 · 배포 단독 수행
  • 데이터를 행동으로 번역 — 보행 신호 잔여 초를 그대로 보여주지 않고, 예상 횡단 시간과 비교해 “지금 건너세요 / N초 후 보행 신호”라는 행동 판단으로 변환하는 것이 제품 원칙
  • 사용자 성향의 제품화 — 안전 우선(버퍼 5분) / 균형(3분) / 정시 우선(1분)의 도착 전략 3모드를 실시간 비교·전환
  • 개인화 학습 — 키 기반 보폭 추정에서 시작해 GPS 실측 보행 속도를 학습, 샘플 10개부터 실측 우선 적용
  • 엔지니어링 품질 — ETA 산식·GPS 투영·신호 파싱을 순수 함수로 분리해 vitest 59건 + Playwright E2E로 테스트, API 키는 전부 서버리스 프록시 뒤에서 보호
